# Operating Playbook

## Purpose

Turn Engram strategy into repeatable, low-friction execution for pilot teams.

## Weekly Loop

1. Review current state.
2. Select top 3 priorities.
3. Assign owners and deadlines.
4. Execute.
5. Capture outcomes and lessons.

Weekly top-3 default for v0.1:
1. Resolve all failed validation checks.
2. Refresh the highest-impact stale canonical pages.
3. Close at least one retrieval correction loop and verify re-indexing.

## Meeting Rhythm

- Planning meeting: Monday (30 min) to set top 3 and assign owners.
- Mid-week check: Wednesday (15 min) focused on blockers and validation failures.
- Retrospective: Friday (30 min) with KPI review and process updates.

## Task Intake Rules

- What gets accepted: tasks that improve canonical context quality, retrieval reliability, or pilot KPI movement.
- What gets deferred: optimizations that do not unblock pilot learning (e.g., advanced ranking or multi-repo federation).
- What gets rejected: work that bypasses write-authority/provenance rules or introduces unreviewed autogenerated canon.

## Execution Standards

- Every task has:
  - owner
  - definition of done
  - risk note
  - rollback option

For Engram tasks, "definition of done" must include:
- validation status (`pass` or accepted exceptions)
- provenance coverage impact
- correction or trust impact notes (if applicable)

## Communication Protocol

- Internal updates: async daily note with changed pages, failed checks, and correction actions.
- Stakeholder updates: weekly one-pager with KPI trend and major decisions.
- Escalation channel: designate a pilot owner + engineering manager for policy conflicts and scope changes.

## Incident Handling

When a critical issue appears:

1. Stabilize.
2. Contain blast radius.
3. Record timeline.
4. Apply fix.
5. Add prevention rule.

Engram-specific critical issue examples:
- Recalled context without provenance used in production decision.
- Canonical page silently overwritten outside approved flow.
- Validation disabled or bypassed for high-risk change path.
